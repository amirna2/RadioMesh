Detailed Description

This class is an interface for a device.

It provides the structure for getting various components of the device, sending data, enabling relay, servicing the interrupt flags and more. Use this inteterface to create specialized devices part of the RadioMesh network.

const std::vector< byte >  data,
std::array< byte, RM_ID_LENGTH >  target = BROADCAST_ADDR 
)
pure virtual

Send data to the mesh network.

A device can send data only if it is included in the network or it is sending one of the inclusion messages:

  • ReservedTopic::INCLUSION_OPEN,
  • ReservedTopic::INCLUSION_REQUEST
  • ReservedTopic::INCLUSION_RESPONSE
  • ReservedTopic::INCLUSION_CONFIRM
Parameters
topicTopic to send the data to
dataData to send
targetTarget device to send the data to (default is broadcast to all devices)
Returns
RM_E_NONE if the data was sent successfully, an error code otherwise.

Implemented in RadioMeshDevice.

Send inclusion open broadcast message.

Hub only feature. Sends an empty broadcast message to notify devices that hub accepts new devices Hub must be in inclusion mode.

Returns
RM_E_NONE on success
RM_E_INVALID_STATE if not in inclusion mode
RM_E_INVALID_DEVICE_TYPE if not a hub

Send inclusion request message.

Device must not be included to send this request. Message contains:

  • Device ID (already in packet header)
  • Device's public key
  • Initial message counter value
Parameters

initialCounterInitial message counter value
Returns

RM_E_INVALID_STATE if device already included
RM_E_INVALID_DEVICE_TYPE if device is a HUB
